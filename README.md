# drupal-issues

Scratchpad for **Drupal.org–ready issue text**: polished summaries, problem statements, and proposed resolutions you can paste into a project queue—without losing the nuance from Slack, IRC, or hallway conversations.

This is not an official Drupal.org repo. It is a **personal workflow repo**: versioned drafts, cross-links between topics, and a running index so the next time someone asks “what did we decide about mcp_server?” you open one file.

---

## Start here

| Where | What |
|--------|------|
| [`issues/INDEX.md`](issues/INDEX.md) | Master list: target project, filename, one-line intent |
| [`issues/`](issues/) | Numbered drafts (`01-…`, `02-…`, …) ready to adapt and file |
| [`community/`](community/) | Extra context (releases, side threads)—not always tied to a single issue |

Before filing on Drupal.org: **retitle**, pick the right **component** and **priority**, add **@mentions** and links to real user profiles, and trim anything that was only for your notes.

---

## What’s inside (at a glance)

MCP consolidation (**mcp_server** vs **mcp_remote**), **ai_providers_api** positioning, **ai_agents_canvas** migration notes, contribution visibility—and occasional **Drupal.org infrastructure** items (for example api.drupal.org / CDN behavior) when they show up in community chat.

The index file holds **chat takeaways** in short form so the table does not have to carry every detail.
