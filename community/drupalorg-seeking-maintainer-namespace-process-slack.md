# Drupal.org — taking over a **seeking maintainer** project namespace (Slack)

**Context:** A contributor asked whether there is a **fast track** to claim an existing **project namespace** on Drupal.org when the project shows **seeking maintainer**, **low site usage** (e.g. **one** active install in **Site information**), and they already filed the **issue queue** request per policy. They need the **namespace** before a **large rename** so work is not blocked.

## Process (from chat — verify on Drupal.org)

- **There is a process to follow** — no separate “express lane” beyond doing it correctly: **project issue** (as already opened), **contact the listed maintainer(s)** through normal channels.
- **Contact surfaces** people used in-thread: **Drupal.org contact form**, **Slack**, **LinkedIn** (and similar) — use whatever reaches a human without bypassing the official steps.
- **Timeline (community recollection, not policy text):** either wait until the **current maintainer agrees** / responds, or — **~two weeks** after outreach if unresponsive (**IIRC** in chat; **confirm** against current **Abandoned projects** / **Project applications** documentation and any staff reply on your request issue).
- **Why it stays slow:** **mcdruid** — the friction is **deliberate**; easy **namespace / maintainer transfers** would echo **WordPress plugin takeover** risks. Expect **PITA**, and treat that as **by design** for **ecosystem safety**.
